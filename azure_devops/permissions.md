# Permissions
Document covers everything related to Permissions

# How to access Permissions?
Project Settings in bottom left -> Permissions (under General)

## Project Adminstrators
- Person accountable for the Project.  Anything that happens in the Project, he/she is responsible that includes Audit.
- Having more than 3 is not recommended.
- Remove users from this group unless they are Product/Project owners.

> _With Great Power comes Great Responsibilities_

## Build Administrators
- Has access to modify the library groups.

## Contributors
- By default all team members are added to this group.

## Readers
- Read-Only access to the Project. 

## Endpoint Administrators
- Has access to create service connections.
- For example, SSH service connection
